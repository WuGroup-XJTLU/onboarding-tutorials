Python for Scientific Computing

Python is the group's glue language: it reads your simulation output, does the arithmetic, and draws the figures. This module covers just enough of the language, then the two libraries you'll use in every later exercise — numpy for numbers and matplotlib for plots.

Learning objectives

  • Read, run and structure Python (types, control flow, functions)
  • Use numpy arrays for numerical data
  • Read and write data files
  • Make clear, labelled matplotlib figures and save them
  • Write small analysis code you can re-run and trust

Prerequisites

  • Module 01 — Unix/Linux Basics (the shell, running files)
  • Module 02 — Bash Scripting (helpful)
  • To install numpy/matplotlib, see Module 05 — Scientific Software & Environments

Estimated time

  • 20–25 hours

Overview

Almost every result the group produces passes through Python at some point: parsing a trajectory, averaging a column, fitting a slope, plotting energy against time. You do not need to become a software engineer — you need to read data reliably, compute the right numbers, and show them clearly. That is what this module builds.

The scientific-Python ecosystem is a small set of libraries that fit together: numpy (arrays and math), matplotlib (plots), scipy (fitting, integration, statistics), pandas (tabular data), and domain tools like MDAnalysis and scikit-learn that you'll meet in later modules. numpy and matplotlib are the foundation the rest stand on, so they are the focus here.

1 · Language essentials

Python is dynamically typed and reads almost like pseudocode; indentation (not braces) defines blocks. The core types you'll use constantly are numbers, strings, lists, and dictionaries:

temperature = 298.0            # float
name = "solvent box"           # string
steps = [100, 200, 400]        # list (ordered, mutable)
params = {"dt": 1.0, "T": 298} # dict: key → value

for s in steps:                # a block is defined by indentation
    print(s, "steps")

if params["T"] > 273:
    print("above freezing")

Factor repeated work into functions, and run code either as a script (python analyse.py) or interactively in a Jupyter notebook for exploration:

def celsius(kelvin):
    """Convert a temperature from kelvin to celsius."""
    return kelvin - 273.15

print(celsius(298.0))          # 24.85

Notebooks to explore, scripts to keep

Jupyter notebooks are excellent for trying things out. But a result you rely on should live in a plain .py script under version control (Module 03) — a notebook run out of order can produce numbers you can't reproduce.

2 · numpy: arrays and vectorised math

A numpy array holds many numbers and lets you operate on them all at once — faster and clearer than looping element by element. This vectorised style is how nearly all scientific Python is written:

import numpy as np

x = np.array([0.0, 1.0, 2.0, 3.0])
y = x**2 + 1                    # applies to every element → [1, 2, 5, 10]

print(y.mean(), y.std())       # reductions over the array
print(x[1:3])                  # slicing → [1., 2.]
print(y[y > 3])                 # boolean masking → [5., 10.]

Arrays can be 2-D (rows and columns), which is how a table of data or a set of coordinates is stored — an N×3 array of atomic positions, say. Operations like arr.mean(axis=0) collapse along one axis.

3 · Reading and writing data

Most simulation output is columns of numbers in a text file. numpy reads and writes these directly:

# a file energy.dat with two columns: time  energy
data = np.loadtxt("energy.dat")      # → 2-D array, one row per line
time   = data[:, 0]                   # first column
energy = data[:, 1]                   # second column

np.savetxt("summary.dat", np.column_stack([time, energy]),
           header="time energy")      # write columns back out

For messier or labelled tables (mixed types, headers, missing values), pandas (pd.read_csv) is worth learning next — but for clean numeric columns, np.loadtxt is all you need.

4 · Plotting with matplotlib

A figure is how you actually see whether a simulation equilibrated or a fit is sensible. The everyday pattern: create axes, plot, label, save.

import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.plot(time, energy, label="potential energy")
ax.set_xlabel("time / ps")            # ALWAYS label axes, with units
ax.set_ylabel("energy / kJ mol$^{-1}$")
ax.legend()
fig.savefig("energy.svg", bbox_inches="tight")   # save, don't just show()

A figure without axis labels is not a result

Label both axes with quantities and units, add a legend when there's more than one series, and save the figure to a file (SVG or PNG) so it can be committed and reused — don't rely on an interactive window. These same plots are the deliverables in the scientific modules.

5 · Analysis you can trust

The habits that keep computational results honest are small:

  • Put reusable work in functions with clear names and a one-line docstring.
  • Sanity-check on a tiny input whose answer you can work out by hand — the same discipline Module 07 — Working with AI Coding Agents applies to generated code.
  • Keep scripts under version control (Module 03) so the exact code behind a figure is recoverable.
  • Watch your units and array shapes — most scientific-Python bugs are a wrong axis, a mismatched length, or a factor you forgot.

Exercise / Deliverable

  1. Write a Python script that reads a two-column data file (e.g. time and energy — reuse simulation output, or generate a noisy sine with numpy).
  2. Compute a summary with numpy: the mean and standard deviation of one column, and the slope of a straight-line fit (np.polyfit).
  3. Put the summary in a function and check it on a tiny hand-computable input.
  4. Produce a matplotlib figure with both axes labelled (with units) and a legend, and save it to a file.

Deliverable: the script, the saved figure, and one sentence stating the value you checked by hand and that the function reproduced it.
