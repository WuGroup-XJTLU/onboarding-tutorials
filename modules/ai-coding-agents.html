Working with AI Coding Agents

AI coding agents can draft scripts, refactor analysis code, and explain unfamiliar tools in seconds. They are powerful assistants and confident liars in equal measure — this module is about using them to go faster without letting a plausible-looking mistake into your science.

Learning objectives

  • Install and run Claude Code and Codex
  • Understand what an "agent" does that a chat box doesn't
  • Write prompts that get useful results
  • Review and verify generated code before trusting it
  • Keep AI-assisted work reproducible and safe with data

Prerequisites

  • Module 01 — Unix/Linux Basics
  • Module 02 — Bash Scripting
  • Module 03 — Version Control with Git
  • Some coding experience (any language) — e.g. Module 04 — Python for Scientific Computing

Estimated time

  • 12–16 hours

Overview

A coding agent is a command-line (or in-editor) tool that can read your project, edit files, run commands, see the results, and iterate — not just answer a question in a chat window. In practice you describe a goal, it proposes and makes changes, runs the code, and reports back; you review, correct, and repeat. Two agents the group uses are Claude Code (Anthropic) and Codex (OpenAI).

The one rule that matters

You are responsible for everything you commit, whether or not an agent wrote it. In research, a script that looks right but is subtly wrong is more dangerous than no script at all. Treat generated code as a capable draft from a colleague who never says "I'm not sure" — read it, test it, and confirm the result is correct before you rely on it.

Installation & setup

Both tools are installed from the command line and need Node.js (a recent LTS version). The exact commands change over time — the official docs linked below are the source of truth — but as of writing:

Claude Code

# install (Node.js 18+)
npm install -g @anthropic-ai/claude-code

# then, inside a project directory:
cd my-project
claude                 # first run walks you through signing in

Authenticate with your Anthropic account (or an API key) when prompted. Run claude in the root of a git repo so it can see your files; ask it for a plan before letting it make changes. The official Claude Code docs are the source of truth for the current install command.

Codex

# install
npm install -g @openai/codex      # (or: brew install codex)

# then, inside a project directory:
codex                  # sign in with your ChatGPT account or an API key

The official Codex documentation is the source of truth for the current install command and sign-in options.

Before you point an agent at a repo

Work inside version control so every change is a reviewable diff you can undo (git init if needed — see Module 03). Never paste API keys or passwords into a prompt, and think before sharing unpublished data or manuscripts with an external service — check with your supervisor about what may leave the group.

The agent loop

Prompting well

  • Give context. Point at the relevant files, describe the data format, state the language/library and versions.
  • Be specific about the goal and constraints. "Write a Python function that reads this XYZ file and returns an N×3 numpy array; no external deps beyond numpy" beats "parse my file".
  • Ask for a plan first on anything non-trivial, then approve or adjust it before code is written.
  • Work in small steps. One change, review, then the next — not a thousand-line rewrite you can't check.
  • Show an example of the input and the expected output when you can.

Reviewing & verifying output

This is the part that separates useful AI assistance from silent errors in your results.

  • Read every diff. If you don't understand a change, ask the agent to explain it — or don't accept it.
  • Test against something you know. Run the code on a tiny input whose answer you can compute by hand.
  • Check the science, not just that it runs. Correct units, correct physics, correct edge cases — code that executes cleanly can still be wrong.
  • Watch for hallucinations. Agents invent plausible function names, library APIs, and even citations. Verify that referenced functions and papers actually exist.
  • Keep a human in the loop for anything that touches data or results.

Reproducibility & attribution

Commit AI-assisted code the same way as any other: small, reviewed commits with clear messages. Keep the important prompts and decisions somewhere in the repo (a note, a commit message) so the reasoning survives. Follow your supervisor's and any publisher's guidance on disclosing AI assistance in papers and code.

Exercise / Deliverable

Pick a small, real task — e.g. "read a directory of simulation output files and plot a quantity vs. time". Then:

  1. Use Claude Code or Codex to produce the script, asking for a plan first and working in small steps.
  2. Verify it: run it on a tiny hand-checkable input, confirm the numbers and units, and note at least one thing you checked that "it runs" would not have caught.
  3. Find (or deliberately probe for) one mistake or questionable choice the agent made, and correct it.

Deliverable: the working script plus a short "verification log" — what you asked, what you checked, what was wrong, and how you confirmed the final result is correct. The verification log is the point of the exercise.
